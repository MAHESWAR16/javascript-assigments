how js is telling foo2 is not funciton above case

How JS knows foo2 is not a function
During Loading Time (Hoisting):
var foo2 = function() { console.log('edupoly') }
Only the variable declaration happens, not the assignment.
So JavaScript stores:
foo2 ‚Üí undefined
Now at Run Time, when JS executes:
foo2()
What actually happens is:
undefined()    // function call means abc() 
Which means we are trying to call undefined like a function.


JavaScript Rule
Only functions can be called with ().
But here:
foo2 is undefined  ‚Üí not a function
So JavaScript throws the error:
TypeError: foo2 is not a function

Explain what happens during loading time and during runtime.
Step-by-step explanation:

üîπ Loading (Creation) Phase:
The JavaScript engine scans the code before execution.
It hoists variable declarations (not assignments) to the top of their scope.
So, internally, JS treats it like this:
var a; // declared but not assigned
console.log(a);
a = 5;
console.log(a);

üîπ Runtime (Execution) Phase:
First console.log(a) ‚Üí a exists but has no value yet ‚Üí undefined
Then a = 5; ‚Üí the value 5 is assigned to a.
Second console.log(a) ‚Üí now prints 5

output
