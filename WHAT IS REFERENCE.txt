 Copy means new separate value. Reference means two variables point to the same memory.

 Imagine This Real Example

🟢 Case 1: Number (Copy)

You write the number 10 on a paper.

You give your friend a photocopy of that paper.

Now:

You change your copy to 20.

Your friend’s copy is still 10.

Because it was a COPY.

That is how numbers and strings , null , undefined , boolean  works.
copy concept apply to these data types  numbers , boolean , strings , null , undefined  1

Reference CONCEPT APLLY TO THESE DATA TYPES  LIKE FUNCTION , OBJECTS , ARRAY OF OBJECTS , ARRAY.
Case 2: Object (Reference)

Now imagine:

There is only ONE whiteboard in the room.

You and your friend both look at the SAME whiteboard.

If your friend changes 10 → 50
You will also see 50.

Because it’s the SAME board.

That is how objects and arrays work.

example 3 ; 

var a = { x: 10 };
var b = a;

b = { x: 100 };

console.log(a.x);
Your answer: 100 ❌

Correct answer: 10

Why?
Step by step:

a → { x: 10 }
b → same object
Then:

b = { x: 100 }
Now b points to a NEW object.

So:

a → { x: 10 }
b → { x: 100 }
They are different now.

So a.x is still 10.

 A reference means:

Two variables pointing to the same memory location.
They are not copies.
They share the same data.

var ar = [10, 20, 30]
var br = ar   // reference copy

br.push(40)

console.log(ar)  // [10, 20, 30, 40]
console.log(br)  // [10, 20, 30, 40]


Why did ar change?

Because:
br is NOT a new array
br is pointing to the SAME array as ar

Memory looks like this:

ar  ----\
          → [10, 20, 30, 40]
br  ----/

Both point to same object.

Reference = sharing the same memory
